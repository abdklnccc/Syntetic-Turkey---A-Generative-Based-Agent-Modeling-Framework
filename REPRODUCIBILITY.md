# Reproducibility Notes

This repository supports two different reproducibility tasks.

## Reproduce the Saved Baseline Output

Use the committed saved outputs. This does not require OpenAI API calls.

The key saved file is:

- `outputs/agent_trajectories.csv`

The companion baseline artifacts are:

- `outputs/aggregate_candidate_intention.csv`
- `outputs/aggregate_party_preference.csv`
- `outputs/evaluation_summary.json`
- `outputs/first_round_vote_distribution.json`
- `outputs/runoff_vote_distribution.json`
- `outputs/broadcasts.jsonl`
- `outputs/reflections.jsonl`
- `outputs/broadcast_cache/broadcasts.json`

To verify the reported panel shape:

```bash
awk -F, 'NR==1 {print NF}' outputs/agent_trajectories.csv
awk -F, 'END {print NR-1}' outputs/agent_trajectories.csv
```

Expected values:

- Columns: `38`
- Data rows: `11100`

To verify that the current configuration still loads and expands to the reported election-safe tick count:

```bash
python3 -B -c "from loaders.config_loader import load_all_configs; from simulation.tick_engine import build_election_safe_tick_plan; b=load_all_configs(); print(len(b.voter_archetypes), len(b.simulation_ticks), len(build_election_safe_tick_plan(b.simulation_ticks)))"
```

Expected values:

- Voter archetypes: `12`
- Raw event ticks: `35`
- Processed ticks: `37`

## Rerun the Simulation

Rerunning the OpenAI pipeline requires an OpenAI API key in `.env`:

```bash
cp .env.example .env
```

Then edit `.env` and set:

```bash
OPENAI_API_KEY=your_api_key_here
```

Use separate rerun directories to avoid overwriting the saved thesis baseline:

```bash
mkdir -p outputs/reruns/openai_baseline/broadcast_cache
mkdir -p logs/reruns/openai_baseline
mkdir -p db_reruns/openai_baseline

OUTPUTS_DIR=outputs/reruns/openai_baseline \
BROADCAST_CACHE_DIR=outputs/reruns/openai_baseline/broadcast_cache \
LOGS_DIR=logs/reruns/openai_baseline \
DB_DIR=db_reruns/openai_baseline \
NUM_AGENTS=300 \
python3 run.py \
  --provider openai \
  --population-profile baseline_2018_300 \
  --parallel-agents \
  --max-workers 5 \
  --continue-on-agent-error \
  --output-dir outputs/reruns/openai_baseline
```

A small mock-mode smoke test can be run without an API key:

```bash
mkdir -p outputs/reruns/mock_smoke/broadcast_cache
mkdir -p logs/reruns/mock_smoke
mkdir -p db_reruns/mock_smoke

OUTPUTS_DIR=outputs/reruns/mock_smoke \
BROADCAST_CACHE_DIR=outputs/reruns/mock_smoke/broadcast_cache \
LOGS_DIR=logs/reruns/mock_smoke \
DB_DIR=db_reruns/mock_smoke \
python3 run.py --mock --max-agents 5 --max-ticks 5 --output-dir outputs/reruns/mock_smoke
```

## Why Exact Numerical Reproduction Is Not Guaranteed

The saved output reproduces the reported baseline artifacts included in this repository. A live OpenAI rerun generates a new run, not a bitwise replay. Outputs may differ because LLM sampling is stochastic, upstream model snapshots can change, and long multi-agent runs compound small generation differences over ticks.

Estimating run-to-run variance would require repeated complete simulation runs. The committed baseline should therefore be used for inspecting the reported result, while reruns should be treated as new experimental runs generated by the same pipeline.
